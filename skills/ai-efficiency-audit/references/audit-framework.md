# 审计框架 · Audit Framework

Backs Stage 2 of ai-efficiency-audit. Score every task, assign a level, compute
savings with the mandatory discount rules.

## 1. 五维评分 (每维 0–4 分)

| 维度 | 0 分 | 2 分 | 4 分 |
|------|------|------|------|
| 重复度 | 每次都不同 | 月度重复 | 每日/每周固定重复 |
| 规则化 | 全凭经验临场判断 | 有大致套路 | 有明确步骤/模板/SOP |
| 输入输出标准化 | 输入来源杂乱、口头为主 | 半结构化(邮件、聊天记录) | 结构化(表格、固定格式文档) |
| 判断密度(反向) | 几乎每步都要专业判断 | 关键节点需判断 | 基本不需要判断 |
| 出错代价(反向) | 错了影响客户/合规/营收 | 错了内部返工 | 错了改一下就行 |

判断密度与出错代价为反向维度:判断越少、代价越低,得分越高。

## 2. 自动化分级映射

总分 0–20,结合一票条款:

- **总分 ≥16 且 出错代价=4** → L3 全自动
- **总分 12–15,或 ≥16 但出错代价 ≤2** → L2 人审环节
- **总分 7–11** → L1 AI 辅助
- **总分 ≤6** → L0 保留人工

一票条款(覆盖总分):
- 涉及合规审批、资金操作、对外法律承诺 → 最高 L1
- 涉及未经公司批准外发的敏感数据 → L0,并在报告中标注数据合规前提
- 任务本身是"建立信任的人际互动"(谈判、绩效面谈、客户安抚) → L0

## 3. 节省工时计算

每个 L1–L3 任务:

```
理论节省 = 单次耗时 × 频率 × 接管比例
  L1 接管比例 30%,L2 60%,L3 85%(上限,不得上调)

保守口径 = 理论节省 × 0.6   ← 强制折减:学习成本、提示词调试、AI返工
乐观口径 = 理论节省 × 0.9
```

强制规则:
- 报告与方案中,**保守口径在前**,乐观口径只作为区间上沿出现
- L2 任务必须把"人审时间"显式从节省中扣除(默认按原耗时的 15% 计)
- 任何任务的保守节省 <15 分钟/周 → 不进 Top 3,避免方案显得在凑数

## 4. 实施成本分级

- **S(小)**: 现成工具 + 提示词即可,半天内跑通 → 速赢项候选
- **M(中)**: 需要搭工作流/接 API/做模板,1–2 周
- **L(大)**: 需要采购、IT 审批或跨团队改流程 → 进报告但不进首期试点

回本周期 = 实施投入工时 ÷ 周保守节省工时,>8 周的标注"暂缓"。

## 5. 试点设计规则 (Top 3 速赢项)

每个速赢项的试点必须包含:

1. **范围**: 一个人或一类任务,不超过两周
2. **基线**: 试点前记录 1 周真实耗时(可用本审计数据)
3. **成功指标**: 提前写死,如"单次耗时下降 ≥30% 且返工率不升"
4. **质量闸门**: L2 任务明确"谁审、审什么、不过审不出门"
5. **回滚条件**: 指标不达标或出现一次重大错误 → 停用并复盘

试点没跑之前,所有数字都叫"估算";试点之后,才叫"证据"。报告结尾必须有这句话
的等价表述,这是本图纸的立场。
