# 方案模板 · Proposal Template (Manager-Facing Memo)

Backs Stage 4. One page, five sections, in this order. Baseline leads, technology
follows. Below the template is a filled example — match its tone: specific, hedged
where honest, zero buzzwords.

## 模板

```
致:[审批人]                                    [日期]
关于:[团队/角色] AI 提效两周试点申请

一、现状基线
  [团队/我] 每周在 [任务类别] 上投入约 X 小时,其中 [具体任务 1/2/3]
  合计 Z 小时,占比 P%。(数据来源:[一周工时盘点/日历回溯],截至 [日期])

二、提案
  对以下 N 项任务引入 AI 辅助,采用"[AI 产出 + 人工审核]"模式:
  1. [任务] → [具体做法,一句话] (等级 L_,实施成本 S/M)
  2. ...
  以下任务明确不引入 AI:[L0 任务及一句话理由]

三、预期收益(保守口径)
  每周节省 A–C 小时(保守 A,乐观 C),相当于 [一个具体的对照物,
  如"每周多出 X 个深度工作时段"]。
  关键假设:[1–2 条,如"任务输入格式保持现状"]

四、风险与回滚
  数据:仅使用 [公司已批准工具/脱敏数据],不涉及 [客户敏感信息等]。
  质量:所有对外产出经 [谁] 审核后发出。
  回滚条件:试点期内 [指标] 未达 [阈值] 或出现一次 [重大错误定义],
  立即停用并恢复原流程。

五、试点计划
  范围:[一人/一类任务] × 两周([起止日期])
  成功指标:[提前写死的可测指标]
  投入:[实施工时,如"首日半天配置"]
  两周后提交试点数据复盘,由您决定是否扩大。
```

## 示例(运营专员,周报+会议纪要场景)

```
致:王总                                        2026-06-15
关于:运营组 AI 提效两周试点申请

一、现状基线
  我每周在例行文档上投入约 6.5 小时:周报整理 2 小时、三场会议纪要
  合计 2.5 小时、竞品动态摘要 2 小时,占每周工时约 16%。
  (数据来源:6/8–6/12 工时盘点)

二、提案
  对以下 3 项引入 AI 辅助,全部采用"AI 初稿 + 本人审核"模式:
  1. 会议纪要 → 录音转写后由 AI 按固定模板出初稿,本人校对(L2,成本 S)
  2. 竞品摘要 → AI 按监测清单生成日摘要,本人筛选入报(L2,成本 S)
  3. 周报 → AI 汇总本周已有记录生成初稿,本人补充判断部分(L1,成本 S)
  不引入 AI:客户群运营答疑 —— 涉及客户信任与未脱敏信息,保留人工。

三、预期收益(保守口径)
  每周节省 2.4–3.6 小时(保守 2.4)。按保守口径,相当于每周多出
  两个完整的专注时段用于活动策划。
  关键假设:会议照常录音;竞品监测清单维持现有 8 个来源。

四、风险与回滚
  数据:仅使用公司已开通的 AI 工具,客户名单与价格信息不进入。
  质量:全部产出经本人审核后才分发。
  回滚条件:两周内出现一次纪要关键信息错漏,或单次校对时间超过
  原人工耗时的 50%,立即停用并复盘。

五、试点计划
  范围:仅本人,6/16–6/27 两周。
  成功指标:三类文档单次耗时平均下降 ≥30%,且未触发回滚条件。
  投入:首日半天完成模板与提示词配置。
  6/30 提交试点数据复盘,由您决定是否在组内推广。
```

## 写作红线

- 不出现"颠覆""革命""全面拥抱 AI"类措辞;具体动词 + 数字
- 不承诺人力替代;收益一律表述为"时间释放到 [更高价值事项]"
- 没做基线盘点就来要方案的,先回 Stage 1,不得编造基线
